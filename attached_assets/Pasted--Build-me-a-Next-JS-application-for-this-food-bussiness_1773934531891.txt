## PHASE 1: HARD PARTS FIRST - SUPABASE INTEGRATION & SECURITY

### Database Schema (Supabase Tables):
1. **menu_items** - id, name, category, is_veg, price, description, image_url, created_at, updated_at
2. **user_submissions** - id, user_name, image_url, experience_text, approved, created_at, ip_address, location
3. **analytics** - id, visitor_ip, country, city, user_agent, visited_at
4. **stopwatch_winners** - id, user_name, time_stopped, prize, created_at, ip_address
5. **stopwatch_attempts** - id, user_id, attempt_date, ip_address (to track daily limit)
6. **festival_discounts** - id, name, discount_percentage, is_active, is_auto, start_date, end_date, poster_url, created_at
7. **truck_location** - id, current_location, latitude, longitude, updated_at, updated_by
8. **admin_users** - id, email, created_at (Supabase Auth will handle passwords)

### Storage Buckets:
1. **user-submissions** - for user uploaded images
2. **festival-posters** - for discount event posters
3. **menu-images** - for menu item images

### Security Implementation:
1. Image upload validation (file type, size, magic bytes check)
2. SQL injection prevention (parameterized queries)
3. XSS prevention (sanitize inputs)
4. Rate limiting on stopwatch game
5. Admin-only routes protection
